# HR信息生命周期管理系统 - 招聘流程管理定义文档

## 1. 文档概述

### 1.1 文档目的
本文档定义了HR信息生命周期管理系统中招聘流程管理的完整生命周期，包括流程阶段、数据结构、业务规则和管理规范。

### 1.2 适用范围
本文档适用于企业的所有招聘活动，包括社招、校招、内推等各种招聘渠道和形式。

### 1.3 招聘流程概览
招聘流程管理包含5个核心阶段：
1. 职位发布阶段
2. 简历筛选阶段
3. 面试评估阶段
4. 录用决策阶段
5. 入职准备阶段

## 2. 招聘流程阶段定义

### 2.1 职位发布阶段 (01_职位发布阶段)

#### 2.1.1 流程说明
职位发布阶段是招聘流程的起点，包括招聘需求收集、职位分析、职位发布等环节。

#### 2.1.2 数据结构

| employment_type | VARCHAR | 50 | 是 | 用工类型（全职/兼职/实习） |
| requester_id | VARCHAR | 50 | 是 | 需求提出人ID |

**职位发布表 (Job Posting)**
| 字段名 | 数据类型 | 长度 | 必填 | 说明 |
|--------|----------|------|------|------|
| posting_id | VARCHAR | 50 | 是 | 发布ID（主键） |
| requirement_id | VARCHAR | 50 | 是 | 需求ID |
| posting_title | VARCHAR | 200 | 是 | 发布标题 |
| posting_content | TEXT | - | 是 | 发布内容 |

#### 2.1.3 业务流程
1. **需求收集**：各部门提交招聘需求
2. **需求审核**：HR审核需求的合理性和紧急程度
3. **需求审批**：上级领导审批招聘需求
4. **职位分析**：HR分析职位要求和市场情况
5. **职位发布**：在多个渠道发布招聘信息
6. **效果监控**：监控发布效果和申请情况

#### 2.1.4 业务规则
- 招聘需求必须经过部门负责人和HR审批
- 薪资范围必须在公司薪酬体系范围内
- 职位发布内容不得包含歧视性条款
- 紧急需求需要在24小时内响应

### 2.2 简历筛选阶段 (02_简历筛选阶段)

#### 2.2.1 流程说明
简历筛选阶段是对收到的简历进行初步筛选和评估的过程。

| expected_salary | VARCHAR | 100 | 否 | 期望薪资 |

**简历筛选标准表 (Screening Criteria)**
| 字段名 | 数据类型 | 长度 | 必填 | 说明 |
|--------|----------|------|------|------|
| criteria_id | VARCHAR | 50 | 是 | 标准ID（主键） |
| posting_id | VARCHAR | 50 | 是 | 职位发布ID |

#### 2.2.3 业务流程
1. **简历收集**：从各渠道收集候选人简历
2. **简历解析**：系统自动解析简历关键信息
3. **初步筛选**：HR根据筛选标准进行初步筛选
4. **技术评估**：技术岗位进行技术能力评估
5. **筛选结果**：记录筛选结果和原因
6. **候选人通知**：通知筛选通过的候选人

#### 2.2.4 业务规则
- 简历筛选应在收到简历后48小时内完成
- 不符合基本条件的简历应立即淘汰
- 优秀简历可以推荐给多个相关职位
- 筛选结果需要记录详细原因
- 禁止基于种族、性别等歧视性因素筛选

面试评估阶段是对筛选通过的候选人进行深入评估的过程。

| updated_time | DATETIME | - | 否 | 更新时间 |

**面试评估表 (Interview Evaluation)**
| 字段名 | 数据类型 | 长度 | 必填 | 说明 |
|--------|----------|------|------|------|
| evaluation_id | VARCHAR | 50 | 是 | 评估ID（主键） |

#### 2.3.3 业务流程
1. **面试安排**：HR协调候选人和面试官时间
2. **面试通知**：发送面试邀请和相关信息
3. **面试准备**：面试官准备面试问题和评估标准
4. **面试执行**：按照预定时间进行面试
5. **面试评估**：面试官填写评估表
6. **评估汇总**：HR汇总所有面试官的评估
7. **结果决策**：决定是否进入下一轮或录用

#### 2.3.4 业务规则
- 面试官必须提前准备面试问题和评估标准
- 每轮面试时间控制在30-60分钟
- 面试评估应在面试后24小时内完成
- 不同级别的职位需要不同轮次的面试
- 面试官需要接受面试技巧培训

| decision_time | DATETIME | - | 是 | 决策时间 |

2. **薪资谈判**：与候选人进行薪资谈判
3. **决策会议**：召开录用决策会议
4. **录用审批**：按照审批链进行录用审批
5. **发送Offer**：向候选人发送正式录用通知
6. **跟进确认**：跟进候选人的接受情况

#### 2.4.4 业务规则
- 录用决策需要多轮审批
- 薪资水平需要符合公司薪酬体系
- 录用通知有效期一般为7天
- 重要岗位需要背景调查
- 高级职位需要高层领导批准

### 2.5 入职准备阶段 (05_入职准备阶段)

#### 2.5.1 流程说明
入职准备阶段是新员工正式入职前的准备工作。

#### 2.5.2 数据结构

**入职准备清单表 (Onboarding Checklist)**
| 字段名 | 数据类型 | 长度 | 必填 | 说明 |

| employee_id | VARCHAR | 50 | 否 | 员工ID（入职后生成） |
| checklist_items | TEXT | - | 是 | 清单项（JSON格式） |
| completion_status | VARCHAR | 50 | 是 | 完成状态（未开始/进行中/已完成） |
| responsible_person | VARCHAR | 50 | 是 | 负责人ID |
| target_completion_date | DATE | - | 是 | 目标完成日期 |
| actual_completion_date | DATE | - | 否 | 实际完成日期 |
| notes | TEXT | - | 否 | 备注 |

**入职材料表 (Onboarding Documents)**
| 字段名 | 数据类型 | 长度 | 必填 | 说明 |
|--------|----------|------|------|------|
| document_id | VARCHAR | 50 | 是 | 材料ID（主键） |
| checklist_id | VARCHAR | 50 | 是 | 清单ID |
| document_type | VARCHAR | 100 | 是 | 材料类型 |
| document_name | VARCHAR | 200 | 是 | 材料名称 |
| is_required | BOOLEAN | - | 是 | 是否必需 |
| document_file | VARCHAR | 500 | 否 | 材料文件路径 |
| submission_status | VARCHAR | 50 | 是 | 提交状态（待提交/已提交/已审核/不合格） |
| review_comments | TEXT | - | 否 | 审核意见 |
| upload_time | DATETIME | - | 否 | 上传时间 |
| review_time | DATETIME | - | 否 | 审核时间 |

#### 2.5.3 业务流程
1. **发送入职指引**：向新员工发送入职指引和准备清单
2. **材料收集**：收集新员工的入职材料
3. **设备准备**：为新员工准备办公设备和账号
4. **工位安排**：安排新员工的工位
5. **培训安排**：安排入职培训和部门介绍
6. **导师分配**：为新员工分配工作导师
7. **入职确认**：确认入职前的所有准备工作

#### 2.5.4 业务规则
- 入职准备工作应在入职前3天完成
- 必需材料不齐全不得办理入职
- 设备和账号应提前1天准备就绪
- 导师应提前确认并做好带教准备
- 入职培训应在新员工入职第一周完成

## 3. 招聘数据分析

### 3.1 关键绩效指标 (KPI)

#### 3.1.1 招聘效率指标
- **招聘周期**：从职位发布到候选人入职的平均天数
- **简历转化率**：面试人数/简历总数的比率
- **面试通过率**：录用人数/面试人数的比率
- **Offer接受率**：接受Offer人数/发送Offer人数的比率

#### 3.1.2 招聘质量指标
- **新员工留存率**：新员工在试用期内留存的比例
- **新员工绩效**：新员工试用期内绩效评估结果
- **用人部门满意度**：用人部门对新员工满意度调查
- **招聘成本**：每个招聘职位的平均成本

#### 3.1.3 招聘渠道指标
- **渠道有效性**：各招聘渠道的简历质量和转化率
- **渠道成本**：各招聘渠道的成本效益分析
- **渠道覆盖**：各招聘渠道的覆盖人群和地域

### 3.2 数据统计报表

#### 3.2.1 招聘漏斗报表
展示从职位发布到入职的各环节数据和转化率。

#### 3.2.2 招聘成本报表
统计招聘过程中的各项费用和成本分析。

#### 3.2.3 招聘渠道报表
分析各招聘渠道的效果和投资回报率。

#### 3.2.4 招聘周期报表
统计不同职位和级别的招聘周期。

- 填写面试评价

## 6. 合规性要求

### 6.1 法律法规
- 遵守《劳动合同法》相关规定
- 不得有就业歧视
- 保护候选人隐私信息
- 规范录用通知和合同

### 6.2 公司政策
- 遵守公司招聘管理制度
- 执行薪酬福利政策
- 遵循审批权限规定
- 维护公司形象和声誉

## 7. 质量保证

### 7.1 流程监控
- 实时监控招聘进度
- 跟踪关键节点
