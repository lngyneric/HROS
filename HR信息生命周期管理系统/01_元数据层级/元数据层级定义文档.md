# HR信息生命周期管理系统 - 元数据层级定义文档

## 1. 文档概述

### 1.1 文档目的
本文档定义了HR信息生命周期管理系统中元数据层级的结构、内容和管理规范，为系统建设和数据管理提供标准化指导。

### 1.2 适用范围
本文档适用于HR信息生命周期管理系统的所有元数据管理活动，包括非敏感信息和敏感信息的分类、存储和管理。

### 1.3 参考依据
- HR系统人事管理菜单结构
- 人事管理业务流程图
- 数据安全法规和最佳实践

## 2. 元数据层级架构

### 2.1 层级结构定义

```
01_元数据层级
├── 01_非敏感信息
│   ├── 01_组织架构元数据
│   ├── 02_职位描述元数据
│   ├── 03_工作流程元数据
│   └── 04_表单模板元数据
└── 02_敏感信息
    ├── 01_员工个人信息
    ├── 02_薪资福利信息
    ├── 03_绩效评估信息
    └── 04_医疗健康信息
```

### 2.2 分类原则

#### 2.2.1 非敏感信息特征
- 可公开或半公开的组织信息
- 不涉及个人隐私的数据
- 用于业务流程和系统配置的基础数据
- 访问权限相对宽松

#### 2.2.2 敏感信息特征
- 涉及个人隐私的数据
- 需要严格权限控制的信息
- 需要加密存储和传输的数据
- 符合数据保护法规要求的信息

## 3. 非敏感信息元数据定义

### 3.1 组织架构元数据 (01_组织架构元数据)

| created_by | VARCHAR | 50 | 是 | 创建人 |
| created_time | DATETIME | - | 是 | 创建时间 |
| updated_by | VARCHAR | 50 | 否 | 更新人 |
| updated_time | DATETIME | - | 否 | 更新时间 |

**职位架构表 (Position)**

|--------|----------|------|------|------|

#### 3.1.3 业务规则
1. 组织架构层级不超过10级
2. 每个组织必须有唯一的组织代码
3. 职位必须归属于某个组织
4. 汇报关系不能形成循环
5. 组织架构变更需要记录历史版本

#### 3.1.4 对应菜单功能
根据人事管理菜单结构，组织架构元数据支持以下功能：
- **组织架构**：查看和维护组织架构树
- **职位架构**：管理职位体系和职位信息
- **人员汇报关系**：设置和管理员工汇报关系
- **基础设置**：组织架构层级主档、层级设定、属性、职位名称主档

#### 3.3.3 人事管理业务流程定义

根据业务流程图，人事管理业务流程包含以下环节：

**1. 预入职雇员输入**
- 流程起点
- 录入即将入职的雇员信息
- 数据流向：→ 雇员资料输入

**2. 雇员资料输入**
- 详细录入雇员的基本资料
- 数据流向：→ 雇员资料变动

**3. 雇员资料变动**
- **单条变动输入**：逐条修改雇员资料
- **批量导入导出**：批量处理雇员资料
- 数据流向：→ 资料变动审核

**4. 资料变动审核**
- 审核雇员资料变动的准确性和合规性
- 数据流向：→ 员工离职处理

**5. 员工离职处理**
- **员工离职处理**：核心离职操作
- **工资计算**：离职员工工资核算
- **修改离职员工状态**：更新员工状态
- 数据流向：→ 离职雇员主档输入

**6. 离职雇员主档输入**
- 录入离职雇员的主档案信息
- 数据流向：→ 员工复职处理

**7. 员工复职处理**
- 处理离职后复职的员工流程
- 完成员工生命周期闭环

## 5. 数据管理规范

### 5.1 数据质量管理

#### 5.1.1 数据完整性要求
1. 必填字段不能为空
2. 外键关联必须有效
3. 数据格式必须符合定义
4. 业务规则必须满足

#### 5.1.2 数据准确性要求
1. 定期进行数据验证和清洗
2. 建立数据纠错机制
3. 重要的数据变更需要审核
4. 保留数据变更历史

#### 5.1.3 数据一致性要求
1. 同一数据在不同表中保持一致
2. 历史数据和当前数据关联清晰
3. 数据状态和实际业务状态同步
4. 系统间数据同步机制完善

### 5.2 数据安全管理

#### 5.2.1 访问控制
1. 基于角色的访问控制（RBAC）
2. 最小权限原则
3. 定期审查访问权限
4. 敏感操作需要审批

#### 5.2.2 数据加密
1. 传输加密：HTTPS/TLS
2. 存储加密：AES-256
3. 密钥管理：定期更换
4. 加密字段：按定义执行

#### 5.2.3 审计日志
1. 记录所有数据访问
2. 记录所有数据修改
3. 记录所有权限变更
4. 日志保留期限：3年

### 5.3 数据生命周期管理
