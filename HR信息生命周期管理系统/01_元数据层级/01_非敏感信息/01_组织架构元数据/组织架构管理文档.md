## 一、组织架构菜单结构

### 1.1 根菜单：组织架构
- **位置**: 界面最顶层的菜单项（左侧列表第一个，带黄色文件夹图标）
- **作用**: 作为整个组织架构模块的入口，聚合所有与组织架构相关的功能

### 1.2 一级子菜单结构

```
根菜单：组织架构

```

## 二、组织架构模块

### 2.1 功能描述
- **功能定位**: 展示组织架构的整体视图（如树形结构、部门/团队层级）
- **用户价值**: 用户查看组织架构的主要入口
- **访问路径**: 组织架构 > 组织架构

### 2.2 核心功能
1. **组织架构树形展示**
   - 可视化展示企业组织结构
   - 支持多层级展开和折叠
   - 显示部门、团队、岗位的层级关系

2. **组织架构搜索**
   - 支持按部门名称搜索
   - 支持按部门编码搜索
   - 支持按负责人姓名搜索

3. **组织架构详情**
   - 部门基本信息（部门名称、编码、类型）
   - 部门负责人信息
   - 部门人员数量统计
   - 部门职责描述

### 2.3 数据结构

#### 部门基本信息表
| 字段名 | 字段类型 | 字段长度 | 是否必填 | 说明 |
|--------|----------|----------|----------|------|
| dept_id | VARCHAR | 50 | 是 | 部门唯一标识 |
| dept_name | VARCHAR | 100 | 是 | 部门名称 |
| dept_code | VARCHAR | 20 | 是 | 部门编码 |
| parent_dept_id | VARCHAR | 50 | 否 | 上级部门ID |
| dept_level | INT | - | 是 | 部门层级 |
| dept_type | VARCHAR | 20 | 是 | 部门类型 |
| dept_manager_id | VARCHAR | 50 | 否 | 部门负责人ID |
| dept_manager_name | VARCHAR | 50 | 否 | 部门负责人姓名 |
| employee_count | INT | - | 否 | 部门人数 |
| create_date | DATETIME | - | 是 | 创建日期 |
| status | CHAR | 1 | 是 | 状态（A-有效/I-无效） |

## 三、职位架构模块

### 3.1 功能描述
- **功能定位**: 管理职位体系，包括职位的定义、层级、职责等
- **用户价值**: 为HR管理提供职位管理的基础功能
- **访问路径**: 组织架构 > 职位架构

### 3.2 核心功能
1. **职位体系管理**
   - 职位分类管理（管理类、技术类、行政类等）
   - 职位等级设置（初级、中级、高级、专家等）
   - 职位职责定义

2. **职位关系管理**
   - 职位上下级关系
   - 职位晋升路径
   - 职位横向流动关系

3. **职位能力要求**
   - 学历要求
   - 工作经验要求
   - 专业技能要求
   - 证书资质要求

### 3.3 数据结构

#### 职位基本信息表
| 字段名 | 字段类型 | 字段长度 | 是否必填 | 说明 |
|--------|----------|----------|----------|------|
| position_id | VARCHAR | 50 | 是 | 职位唯一标识 |
| position_name | VARCHAR | 100 | 是 | 职位名称 |
| position_code | VARCHAR | 20 | 是 | 职位编码 |
| position_category | VARCHAR | 50 | 是 | 职位分类 |
| position_level | VARCHAR | 20 | 是 | 职位等级 |
| dept_id | VARCHAR | 50 | 是 | 所属部门ID |
| position_type | VARCHAR | 20 | 是 | 职位类型 |
| education_requirement | VARCHAR | 100 | 否 | 学历要求 |
| experience_requirement | VARCHAR | 200 | 否 | 经验要求 |
| skill_requirement | TEXT | - | 否 | 技能要求 |
| create_date | DATETIME | - | 是 | 创建日期 |
| status | CHAR | 1 | 是 | 状态（A-有效/I-无效） |

## 四、人员汇报关系模块

### 4.1 功能描述
- **功能定位**: 查看或管理员工的汇报关系（如上级、下级、跨部门汇报等）
- **用户价值**: 组织架构中"人员与职位关联"的重要模块
- **访问路径**: 组织架构 > 人员汇报关系

### 4.2 核心功能
1. **汇报关系展示**
   - 上级汇报关系
   - 下级管理关系
   - 虚线汇报关系（跨部门协作）

2. **汇报关系管理**
   - 设置直接上级
   - 设置虚线汇报对象
   - 批量调整汇报关系

3. **汇报关系查询**
   - 按员工查询汇报关系
   - 按部门查询汇报关系
   - 按职位查询汇报关系

### 4.3 数据结构

#### 人员汇报关系表
| 字段名 | 字段类型 | 字段长度 | 是否必填 | 说明 |
|--------|----------|----------|----------|------|
| relation_id | VARCHAR | 50 | 是 | 关系唯一标识 |
| employee_id | VARCHAR | 50 | 是 | 员工ID |
| supervisor_id | VARCHAR | 50 | 是 | 上级ID |
| relation_type | CHAR | 1 | 是 | 关系类型（1-直接汇报/2-虚线汇报） |
| effective_date | DATE | - | 是 | 生效日期 |
| end_date | DATE | - | 否 | 结束日期 |
| create_date | DATETIME | - | 是 | 创建日期 |
| status | CHAR | 1 | 是 | 状态（A-有效/I-无效） |

## 五、基础设置模块

### 5.1 组织架构层级主档

#### 功能描述
定义组织架构的**层级结构**（如"总部-事业部-部门-班组"），是组织架构的核心框架。

#### 数据结构
| 字段名 | 字段类型 | 字段长度 | 是否必填 | 说明 |
|--------|----------|----------|----------|------|
| level_id | VARCHAR | 50 | 是 | 层级唯一标识 |
| level_name | VARCHAR | 100 | 是 | 层级名称 |
| level_code | VARCHAR | 20 | 是 | 层级编码 |
| level_order | INT | - | 是 | 层级顺序 |
| level_description | VARCHAR | 500 | 否 | 层级描述 |
| parent_level_id | VARCHAR | 50 | 否 | 上级层级ID |
| create_date | DATETIME | - | 是 | 创建日期 |
| status | CHAR | 1 | 是 | 状态（A-有效/I-无效） |

#### 示例数据
```
1. 总部（Level 1）
2. 事业部（Level 2）
3. 部门（Level 3）
4. 班组（Level 4）
```

### 5.2 组织架构层级设定

#### 功能描述
设置组织架构层级的**具体规则**（如层级数量、层级名称、层级之间的关联关系），用于规范组织架构的构建。

#### 配置项
- **层级数量限制**: 最多允许的组织层级数
- **层级命名规则**: 每个层级的命名规范
- **层级关系规则**: 层级之间的父子关系约束
- **层级权限规则**: 不同层级的权限配置

#### 数据结构
| 字段名 | 字段类型 | 字段长度 | 是否必填 | 说明 |
|--------|----------|----------|----------|------|
| setting_id | VARCHAR | 50 | 是 | 设定唯一标识 |
| setting_name | VARCHAR | 100 | 是 | 设定名称 |
| setting_value | VARCHAR | 500 | 是 | 设定值 |
| setting_description | VARCHAR | 500 | 否 | 设定描述 |
| create_date | DATETIME | - | 是 | 创建日期 |
| update_date | DATETIME | - | 是 | 更新日期 |

### 5.3 组织架构属性

#### 功能描述
配置组织架构的**附加属性**（如部门类型、部门职责、部门负责人权限等），丰富组织架构的信息维度。

#### 数据结构
| 字段名 | 字段类型 | 字段长度 | 是否必填 | 说明 |
|--------|----------|----------|----------|------|
| attribute_id | VARCHAR | 50 | 是 | 属性唯一标识 |
| attribute_name | VARCHAR | 100 | 是 | 属性名称 |
| attribute_code | VARCHAR | 20 | 是 | 属性编码 |
| attribute_type | VARCHAR | 20 | 是 | 属性类型（文本/数字/日期/枚举） |
| attribute_value | VARCHAR | 500 | 否 | 属性值 |
| is_required | CHAR | 1 | 是 | 是否必填（Y/N） |
| create_date | DATETIME | - | 是 | 创建日期 |

#### 示例属性
```
1. 部门类型（成本中心/利润中心）
2. 部门职责（主营业务/支持职能）
3. 部门预算权限（有/无）
4. 部门招聘权限（有/无）
```

### 5.4 职位名称主档

#### 功能描述
定义职位的**标准名称**（如"总经理""部门经理""专员"），确保职位名称的统一性和规范性，避免歧义。

#### 数据结构
| 字段名 | 字段类型 | 字段长度 | 是否必填 | 说明 |
|--------|----------|----------|----------|------|
| title_id | VARCHAR | 50 | 是 | 职位名称唯一标识 |
| title_name | VARCHAR | 100 | 是 | 职位名称 |
| title_code | VARCHAR | 20 | 是 | 职位编码 |
| title_category | VARCHAR | 50 | 是 | 职位分类 |
| title_level | VARCHAR | 20 | 是 | 职位等级 |
| title_description | VARCHAR | 500 | 否 | 职位描述 |
| create_date | DATETIME | - | 是 | 创建日期 |
| status | CHAR | 1 | 是 | 状态（A-有效/I-无效） |

#### 示例数据
```
管理类：
- 总经理
- 副总经理
- 部门经理
- 主管

技术类：
- 技术总监
- 高级工程师
- 工程师
- 助理工程师

行政类：
- 行政经理
- 行政专员
- 前台文员
```

## 七、业务流程

### 7.1 组织架构调整流程
```
1. 发起申请 → 2. 部门审批 → 3. HR审核 → 4. 系统配置 → 5. 通知发布
```

### 7.2 职位新增流程
```
1. 需求分析 → 2. 职位设计 → 3. 部门确认 → 4. HR审核 → 5. 系统录入
```

### 7.3 人员调整流程
```
1. 调整申请 → 2. 上级审批 → 3. HR审核 → 4. 系统更新 → 5. 通知相关方
```

## 九、系统接口

### 9.1 API接口列表
1. **获取组织架构树**: `GET /api/organization/tree`
2. **获取部门详情**: `GET /api/organization/department/{id}`
3. **新增部门**: `POST /api/organization/department`
4. **更新部门**: `PUT /api/organization/department/{id}`
5. **删除部门**: `DELETE /api/organization/department/{id}`
6. **获取职位列表**: `GET /api/organization/positions`
7. **获取汇报关系**: `GET /api/organization/reporting/{employeeId}`

### 9.2 数据导入导出
- **Excel导入**: 支持批量导入组织架构数据
- **Excel导出**: 支持导出完整的组织架构数据
- **JSON导出**: 支持导出结构化的组织架构数据

## 十、注意事项

### 10.1 数据维护
1. 定期检查组织架构的准确性
2. 及时更新部门负责人信息
3. 保持职位名称的规范性
4. 确保汇报关系的正确性

### 10.2 系统使用
1. 组织架构调整需经过审批流程
2. 职位名称必须使用标准名称
3. 部门编码不可重复
4. 删除操作需谨慎，建议使用停用状态

### 10.3 安全管理
1. 敏感信息访问需授权
2. 批量操作需二次确认
3. 重要变更需记录日志
4. 定期备份数据
