## 一、概述

### 1.1 组织架构元数据定义
组织架构元数据是HR信息生命周期管理系统的基础数据，定义了组织架构的结构、层级、职位和汇报关系等核心信息。

### 1.2 元数据的重要性
1. **系统基础**: 为HR系统的其他模块提供基础数据支撑
2. **标准化管理**: 建立统一的组织架构标准和规范
3. **数据一致性**: 确保组织架构数据的一致性和准确性
4. **系统集成**: 为与其他系统的集成提供标准接口

## 四、数据结构概览

### 4.1 核心数据表

#### 部门表（departments）
- 部门基本信息（ID、名称、编码）
- 组织层级信息（层级、上级部门）
- 管理信息（负责人、人数）
- 状态信息（创建日期、状态）

#### 职位表（positions）
- 职位基本信息（ID、名称、编码）
- 职位分类信息（分类、等级）
- 能力要求（学历、经验、技能）
- 关联信息（所属部门、汇报对象）

#### 汇报关系表（reporting_relations）
- 关系信息（员工ID、上级ID）
- 关系类型（直接/虚线/临时）
- 时间信息（生效日期、结束日期）
- 变更信息（变更原因、审批状态）

#### 配置表（settings）
- 配置信息（名称、编码、值）
- 配置分类（层级、属性、职位）
- 配置描述（说明、示例）
- 管理信息（创建时间、状态）

### 4.2 数据关系图
```
departments (部门表)
    ├── parent_dept_id → departments (自关联)
    ├── dept_manager_id → employees (负责人)
    └── positions (职位表)
            ├── dept_id → departments (所属部门)
            └── reporting_relations (汇报关系表)
                    ├── employee_id → employees (员工)
                    └── supervisor_id → employees (上级)
```

## 七、使用指南

### 7.1 快速开始
1. **查看组织架构**: 登录系统 → 组织架构 → 组织架构
2. **查询职位信息**: 组织架构 → 职位架构 → 职位查询
3. **设置汇报关系**: 组织架构 → 人员汇报关系 → 新增关系
4. **配置基础设置**: 组织架构 → 基础设置 → 相应子菜单

### 7.2 常用操作
- **新增部门**: 组织架构 → 新增 → 填写部门信息 → 提交审批
- **调整组织**: 选中部门 → 编辑 → 调整上级或属性 → 保存
- **职位查询**: 职位架构 → 搜索框 → 输入关键词 → 查询
- **变更汇报**: 人员汇报关系 → 选中记录 → 变更 → 填写原因 → 提交

### 7.3 注意事项
1. **数据准确性**: 确保组织架构数据的准确性
2. **及时更新**: 组织调整后及时更新系统
3. **权限管理**: 合理分配系统访问权限
4. **数据备份**: 定期备份组织架构数据

## 九、最佳实践

### 9.1 组织架构设计
1. **层级合理**: 建议控制在3-5级
2. **管理幅度**: 每个管理者直接下属不超过15人
3. **职责清晰**: 部门职责明确，避免重叠
4. **灵活调整**: 根据业务发展适时调整

### 9.2 职位体系设计
1. **分类科学**: 职位分类要科学合理
2. **等级清晰**: 职位等级与能力、薪资匹配
3. **路径明确**: 晋升路径清晰可行
4. **标准统一**: 职位名称标准化

### 9.3 汇报关系设计
1. **唯一性**: 每个员工只有一个直接上级
2. **清晰性**: 汇报关系明确无歧义
3. **灵活性**: 支持矩阵式和项目制管理
4. **可追溯**: 保留历史变更记录
