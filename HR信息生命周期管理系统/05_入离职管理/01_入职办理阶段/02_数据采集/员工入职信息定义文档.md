# 员工入职信息定义文档

## 1. 文档概述

### 1.1 目的
本文档定义了员工入职过程中的信息采集、处理和管理规范，确保入职流程的标准化和数据完整性。

### 1.2 适用范围
适用于所有新员工入职的信息管理，包括数据采集、验证、审核和系统录入等环节。

### 1.3 与元数据的关系
本文档基于《员工信息定义文档》中的数据标准，重点定义入职流程中的数据处理规范。

## 2. 入职流程数据映射

### 2.1 入职前准备阶段

#### 2.1.1 招聘信息传递
| 数据项 | 来源 | 目标系统 | 处理规则 |
|--------|------|----------|----------|
| 应聘者姓名 | 招聘系统 | 入职系统 | 直接传递 |
| 应聘职位 | 招聘系统 | 入职系统 | 需与职位体系匹配 |
| 面试评价 | 招聘系统 | 入职系统 | 仅HR可见 |
| 薪资标准 | 招聘系统 | 薪资系统 | 加密传递 |

#### 2.1.2 预入职信息采集
| 数据项 | 采集方式 | 验证规则 | 完成时限 |
|--------|----------|----------|----------|
| 个人基本信息 | 企微表单 | 格式验证 | 入职前3天 |
| 身份证件 | 上传照片 | OCR识别+人工审核 | 入职前3天 |
| 学历证明 | 上传照片 | 学信网验证 | 入职前3天 |
| 体检报告 | 上传文件 | 格式验证 | 入职前1天 |

### 2.2 入职办理阶段

#### 2.2.1 现场核验信息
| 数据项 | 核验方式 | 异常处理 | 责任人 |
|--------|----------|----------|--------|
| 身份证原件 | 现场核验+复印件留存 | 不一致则终止入职 | HR专员 |
| 学历证书 | 现场核验+复印件留存 | 不一致则标注并确认 | HR专员 |
| 体检报告 | 现场核验+原件留存 | 不合格则终止入职 | HR专员 |
| 离职证明 | 现场核验+原件留存 | 无法提供则背景调查 | HR专员 |

#### 2.2.2 系统信息录入
| 数据项 | 录入系统 | 录入时机 | 审核要求 |
|--------|----------|----------|----------|
| 员工基本信息 | HR系统 | 入职当天 | HR主管审核 |
| 组织架构信息 | HR系统 | 入职当天 | 自动验证 |
| 薪资信息 | 薪资系统 | 入职当天 | 财务审核 |
| 账号权限 | IT系统 | 入职当天 | IT管理员分配 |

### 2.3 入职后完善阶段

#### 2.3.1 试用期信息跟踪
| 数据项 | 更新频率 | 负责人 | 用途 |
|--------|----------|--------|------|
| 试用期评价 | 每月 | 直属主管 | 试用期考核 |
| 培训记录 | 实时 | HR专员 | 培训完成度 |
| 绩效数据 | 实时 | 绩效系统 | 绩效考核 |
| 考勤记录 | 每日 | 考勤系统 | 考勤管理 |

#### 2.3.2 转正信息处理
| 数据项 | 处理时机 | 处理方式 | 后续动作 |
|--------|----------|----------|----------|
| 转正评价 | 试用期结束前1周 | 主管填写 | 转正决策 |
| 转正决策 | 试用期结束前3天 | HR审核 | 系统状态更新 |
| 薪资调整 | 转正当天 | 财务处理 | 薪资系统更新 |
| 合同签署 | 转正后1周内 | 法务处理 | 合同系统更新 |

## 3. 数据字段详细定义

### 3.1 基础信息字段

#### 3.1.1 姓名 (name)
- **数据类型**: VARCHAR(50)
- **必填**: 是
- **验证规则**:
  - 长度2-50个字符
  - 只能包含中文、英文、空格
  - 与身份证姓名必须一致
- **采集时机**: 入职前3天
- **采集方式**: 企微表单填写
- **审核要求**: 与身份证原件核验

#### 3.1.2 性别 (gender)
- **数据类型**: VARCHAR(10)
- **必填**: 是
- **验证规则**:
  - 只能是"男"或"女"
  - 与身份证性别必须一致
- **采集时机**: 入职前3天
- **采集方式**: 企微表单选择
- **审核要求**: 与身份证原件核验

#### 3.1.3 身份证号 (id_number)
- **数据类型**: VARCHAR(18)
- **必填**: 是
- **验证规则**:
  - 18位身份证号码格式
  - 通过校验位验证
  - 系统内唯一性检查
- **采集时机**: 入职前3天
- **采集方式**: 企微表单填写+OCR识别
- **审核要求**: 与身份证原件完全一致
- **安全要求**: 加密存储，显示时脱敏

#### 3.1.4 手机号码 (mobile)
- **数据类型**: VARCHAR(11)
- **必填**: 是
- **验证规则**:
  - 11位手机号码格式
  - 通过短信验证码验证
- **采集时机**: 入职前3天
- **采集方式**: 企微表单填写+短信验证
- **审核要求**: 通过短信验证码确认

#### 3.1.5 邮箱 (email)
- **数据类型**: VARCHAR(100)
- **必填**: 是
- **验证规则**:
  - 标准邮箱格式
  - 通过邮箱验证链接验证
- **采集时机**: 入职前3天
- **采集方式**: 企微表单填写+邮箱验证
- **审核要求**: 通过邮箱验证链接确认

### 3.2 工作信息字段

#### 3.2.1 部门 (department)
- **数据类型**: VARCHAR(100)
- **必填**: 是
- **验证规则**:
  - 必须存在于组织架构中
  - 与招聘意向部门一致
- **采集时机**: 入职当天
- **采集方式**: HR系统选择
- **审核要求**: 与组织架构自动匹配

#### 3.2.2 岗位 (position)
- **数据类型**: VARCHAR(100)
- **必填**: 是
- **验证规则**:
  - 必须存在于职位体系中
  - 与招聘意向岗位一致
- **采集时机**: 入职当天
- **采集方式**: HR系统选择
- **审核要求**: 与职位体系自动匹配

#### 3.2.3 工号 (employee_id)
- **数据类型**: VARCHAR(20)
- **必填**: 是
- **验证规则**:
  - 系统自动生成
  - 格式：SCH + 5位数字
  - 系统内唯一性
- **采集时机**: 入职当天
- **采集方式**: HR系统自动生成
- **审核要求**: 无需人工审核

#### 3.2.4 企微ID (wework_id)
- **数据类型**: VARCHAR(50)
- **必填**: 是
- **验证规则**:
  - 与企业微信后台一致
  - 系统内唯一性
- **采集时机**: 入职当天
- **采集方式**: 企业微信同步
- **审核要求**: 与企业微信后台自动同步

#### 3.2.5 工作地点 (work_location)
- **数据类型**: VARCHAR(100)
- **必填**: 是
- **验证规则**:
  - 必须存在于工作地点主数据中
  - 与部门所在地匹配
- **采集时机**: 入职当天
- **采集方式**: HR系统选择
- **审核要求**: 与部门信息自动匹配

#### 3.2.6 入职日期 (hire_date)
- **数据类型**: DATE
- **必填**: 是
- **验证规则**:
  - 格式：YYYY-MM-DD
  - 不能早于当前日期
  - 不能晚于合同开始日期
- **采集时机**: 入职当天
- **采集方式**: HR系统填写
- **审核要求**: 与劳动合同一致

### 3.3 可选信息字段

#### 3.3.1 民族 (nation)
- **数据类型**: VARCHAR(20)
- **必填**: 否
- **验证规则**:
  - 必须在民族主数据中
  - 与身份证民族一致
- **采集时机**: 入职前3天
- **采集方式**: 企微表单选择
- **审核要求**: 与身份证原件核验

#### 3.3.2 籍贯 (hometown)
- **数据类型**: VARCHAR(100)
- **必填**: 否
- **验证规则**:
  - 格式：省份-城市
  - 与身份证籍贯一致
- **采集时机**: 入职前3天
- **采集方式**: 企微表单选择
- **审核要求**: 无需人工审核

#### 3.3.3 出生日期 (birth_date)
- **数据类型**: DATE
- **必填**: 否
- **验证规则**:
  - 格式：YYYY-MM-DD
  - 与身份证出生日期一致
  - 年龄需满18周岁
- **采集时机**: 入职前3天
- **采集方式**: 从身份证号自动提取
- **审核要求**: 与身份证号自动匹配

#### 3.3.4 居住地址 (address)
- **数据类型**: VARCHAR(200)
- **必填**: 否
- **验证规则**:
  - 详细的居住地址
  - 包含省市区街道门牌号
- **采集时机**: 入职前3天
- **采集方式**: 企微表单填写
- **审核要求**: 无需人工审核

## 4. 数据处理流程

### 4.1 数据采集流程
```
1. 企微表单填写 → 2. 格式验证 → 3. 临时存储 → 4. HR初审 → 5. 现场核验 → 6. 系统录入
```

### 4.2 数据验证流程
```
1. 格式验证 → 2. 业务规则验证 → 3. 唯一性验证 → 4. 一致性验证 → 5. 外部接口验证
```

### 4.3 数据审核流程
```
1. 自动验证 → 2. HR专员审核 → 3. HR主管审核 → 4. 异常处理 → 5. 审核通过
```

## 5. 异常情况处理

### 5.1 数据不一致
- **身份证与填写信息不一致**: 终止入职，要求重新填写
- **学历验证失败**: 背景调查，确认真实性
- **体检不合格**: 根据具体情况决定是否入职

### 5.2 数据缺失
- **必填字段缺失**: 要求补全信息，否则无法进入下一环节
- **可选字段缺失**: 允许后续补充，不影响入职流程

### 5.3 系统异常
- **企微表单提交失败**: 提供备用的线下表格填写
- **HR系统录入失败**: 临时记录，系统恢复后补录
- **外部接口调用失败**: 人工验证，记录异常日志

## 6. 数据安全和隐私保护

### 6.1 敏感信息处理
- **身份证号**: 加密存储，仅HR和法务可查看完整信息
- **家庭住址**: 仅HR和本人可查看
- **学历信息**: 仅HR和直属主管可查看

### 6.2 访问权限控制
- **员工本人**: 可查看和修改基本信息
- **HR专员**: 可查看和修改所有入职信息
- **HR主管**: 可审核和最终确认入职信息
- **IT管理员**: 仅可查看系统相关字段

### 6.3 数据传输安全
- **加密传输**: 所有数据传输使用HTTPS协议
- **接口认证**: 外部接口调用需要身份认证
- **日志记录**: 记录所有数据访问和修改操作
