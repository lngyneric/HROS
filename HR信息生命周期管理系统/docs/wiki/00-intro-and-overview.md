# 00. 引言与总览

## 0.1 仓库定位

- 本仓库是“HR信息生命周期管理系统”的文档与数据资产库，重点沉淀：
  - HR 业务流程结构（招聘、培训、薪资、入离职等）的目录化设计
  - 各类元数据定义（组织架构、职位、流程、表单、员工信息表结构等）
  - 示例或真实的 JSON 数据文件（员工在职/离职信息、部门信息、分析报告等）
  - 少量自动化脚本（当前为入离职场景的数据清洗与归档脚本）
- 仓库当前并不是一个完整的可部署“应用系统”，系统总览文档中提到的前端/后端/数据库/容器化等技术架构属于规划目标，代码尚未落地。

## 0.2 顶层目录结构总览

根目录结构（简化）：

```text
HR信息生命周期管理系统/
├── 01_元数据层级/
├── 02_招聘流程管理/
├── 03_培训与发展管理/
├── 04_薪资管理/              # 目前为规划中的目录，尚未落地到仓库
├── 05_入离职管理/
├── 06_分析数据层级/
├── HR信息生命周期管理系统总览.md
├── README.md
└── 文档索引.md 等说明类文档
```

其中：

- 01_元数据层级：描述整个系统的“基础配置数据”，包括组织架构、职位定义、流程配置、表单模板、员工信息表结构等。
- 02_招聘流程管理：围绕招聘全流程（职位发布→简历筛选→面试评估→录用决策→入职准备）的文档与样例数据。
- 03_培训与发展管理：培训需求分析、计划制定、实施与评估，以及员工发展规划的结构性文档。
- 04_薪资管理：在总览文档中已有清晰规划（核算、发放、福利、税务、分析），但当前仓库中尚未创建对应子目录和定义文档。
- 05_入离职管理：包含入职、在职、离职等阶段的大量员工、部门 JSON 数据，并提供一个 Node.js 脚本用于数据清洗与离职归档。
- 06_分析数据层级：存放分析结果类 JSON，例如员工状态分析报告、新员工入职分析、综合信息分析等，用于支持数据驱动决策。

## 0.4 业务视角的生命周期说明

系统围绕“员工生命周期”组织数据和文档，主要阶段包括：

- 招聘与预入职：职位需求分析、渠道发布、简历筛选、面试评估、录用决策、入职准备等。
- 入职与在职管理：员工档案建立、职位与部门关联、合同与绩效记录、在职期间的各种信息变动。
- 离职与离职后管理：离职申请、面谈、交接、审批，到离职后档案/竞业限制/补偿等管理。
- 培训与发展：培训需求评估、课程与计划安排、培训实施、评估及后续发展规划。
- 薪资与福利：薪资核算、发放、福利与税务处理，以及相关的分析。
- 分析与决策：基于上述阶段产生的大量结构化数据进行分析，支撑 HR 决策。

本仓库通过目录和文档，在“没有完整应用系统代码”的前提下，先行定义和固化上述各阶段所需的元数据结构和数据资产形态。

## 0.5 技术视角的现状与规划

- 现状：
  - 仓库中几乎没有前后端项目代码，也没有 package.json / requirements.txt 等工程化依赖清单。
  - 唯一实际可执行代码是 [05_入离职管理/data_cleaning_and_archive.js](file:///Users/cherrych/Documents/trae_projects/HROS/HR信息生命周期管理系统/05_入离职管理/data_cleaning_and_archive.js)，用于在本地对在职/离职员工 JSON 数据进行清洗、归档和统计。
- 规划（见系统总览的“技术架构”章节）：
  - 前端：React + Ant Design
  - 后端：Node.js + Express
  - 数据库：PostgreSQL
  - 部署：Docker 容器化
  - 安全：JWT + RBAC + 加密存储 + 审计日志

本 Code Wiki 会在后续章节中明确区分“当前实现”与“规划中的技术架构”，避免误认为这些技术栈已经在仓库中落实。

## 0.7 三个世界视角

### 0.7.1 原子世界（有空间 + 有时间）

- 本仓库不直接承载招聘面试、入离职办理、培训授课、发薪打款等现实动作本身。
- 现实动作受物理与组织约束（排期、审批链、资源占用），只能通过流程设计与制度安排来保障可执行。

### 0.7.2 信息世界（无空间 + 有时间）

- 本仓库的核心定位是信息世界的“标准库”：用目录结构、文档与 JSON 数据沉淀可复用的结构与样例。
- “时间”主要体现在信息的版本与变更顺序：字段定义更新、数据清洗与归档、分析报告生成都有先后关系，需要可追溯。

### 0.7.3 AI 世界（无空间 + 存储层无时间）

- AI 世界的前提是信息世界的高质量数据基座：结构统一、历史保留、口径一致。
- 本仓库可以演进为 AI 的知识与数据底座，用于支持离职风险预测、招聘渠道优化、培训效果评估等能力，但这些能力目前仍处于规划阶段。
